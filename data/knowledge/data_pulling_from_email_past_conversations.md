# Workflow: Data pulling from email past conversations

This workflow **pulls the last emails** for a contact, **downloads quote PDFs** from those threads, **extracts machine specs and prices** from the PDFs, and **inserts them into the main email (TO_SEND)** so the recipient sees the latest machines and prices we quoted. Run it before sending a lead email when you need the email to show exact quote data.

## When to use

- Preparing to email a lead and you want the **email body to show the latest machine specs and prices** we sent.
- You need quote data (prices, machine models, key specs) that was sent as PDF attachments — and you want that data **inserted into the TO_SEND email**, not just in a separate file.
- Building a send-ready email that references our last quote with concrete numbers.

## Pipeline (four steps, in order)

### 1. Pull last emails (conversation history + logic tree)

Searches Gmail to/from the contact and writes one markdown file with timeline, proposals we sent, and their feedback.

```bash
poetry run python scripts/pull_contact_email_history.py \
  --email <contact_email> \
  --output data/imports/24_WebSite_Leads/<folder>_email_history.md \
  --store-memory \
  --summarize \
  [--name "Contact Name"]
```

**Output:** e.g. `data/imports/24_WebSite_Leads/forma3d_eduardo_email_history.md` — use this when drafting so you don’t repeat past proposals.

### 2. Download PDFs from those emails

Downloads all PDF attachments from threads involving the contact into a folder. Required so step 3 can extract from them.

```bash
poetry run python scripts/download_email_attachments.py \
  --email <contact_email> \
  --folder <folder_name>
```

**Output:** PDFs in `data/imports/downloaded_from_emails/<folder_name>/` (e.g. `forma3d_eduardo/`).

### 3. Extract specs and prices from the PDFs

Runs PDF text extraction + LLM to pull **all machines, prices, and key specs** from the downloaded PDFs. Writes `quote_summary.md` in the same folder and can update the TO_SEND email (step 4).

```bash
poetry run python scripts/download_email_attachments.py \
  --email <contact_email> \
  --folder <folder_name> \
  --analyze
```

With **--to-send** and **--memory**, the same script also inserts into the email and stores a memory (see step 4).

### 4. Insert into the main email (TO_SEND)

The TO_SEND email template contains a placeholder:

- **`<!-- INSERT_LATEST_QUOTE -->`**

When you run the download script with **--analyze** and **--to-send**, it **replaces that placeholder** with a bullet list of the latest machines, prices, and specs extracted from the PDFs. The email then shows the recipient the exact quote data (e.g. “**PF1-X-1210:** EUR 95,000 — 1200×2000 mm”).

**Full run (steps 2 + 3 + 4 in one go):**

```bash
# Step 1 — pull emails (separate script)
poetry run python scripts/pull_contact_email_history.py \
  --email pinto@forma3d.pt \
  --output data/imports/24_WebSite_Leads/forma3d_eduardo_email_history.md \
  --store-memory --summarize --name "Eduardo Pinto"

# Steps 2–4 — download PDFs, extract, insert into email
poetry run python scripts/download_email_attachments.py \
  --email pinto@forma3d.pt \
  --folder forma3d_eduardo \
  --analyze \
  --to-send data/imports/24_WebSite_Leads/email_lead3_eduardo_pinto_TO_SEND.md \
  [--memory] \
  [--name "Eduardo Pinto"]
```

After this, open the TO_SEND file: the “Latest quote we sent you” section will contain the extracted machines and prices (or a fallback line if extraction found nothing).

## What the email shows

The main email body includes a block like:

**Latest quote we sent you:**

• **Custom-size ATF (roll-fed):** EUR 120,000 — roll-fed, …  
• **PF1-X-1210:** EUR 95,000 — 1200×2000 mm, …  
• **AM roll-fed:** EUR … — …

(This block is generated by the script from the PDFs. If no PDFs or extraction fails, the script inserts a fallback line so you can paste manually or refer to the quote we sent.)
